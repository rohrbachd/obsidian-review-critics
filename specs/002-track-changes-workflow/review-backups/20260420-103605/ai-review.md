# AI Code Review

## Summary

- Full-scan review executed across tracked and untracked repository code, prioritizing runtime safety paths in `src/` and coverage in `tests/`.
- Found one **critical** reliability issue that can permanently block all commands in-session when pane refresh throws.
- Found one **major** correctness issue in `rejectTrackedChangeByMarkup` that can drop original text for deletion/substitution rejection.
- Found one **major** test-gap issue: critical execution paths are currently untested, increasing regression risk in areas that recently had production defects.
- Found one **minor** ID-generation weakness (`Date.now`) that can create duplicate theme preset IDs under fast consecutive saves.

## Findings

| Severity | Location                                                                              | Issue                                                                                                                                                                                                   | Impact                                                                                                                                   | Recommendation                                                                                                                       | Validation                                                                                                                |

| critical | `src/main.ts:703-705`                                                                 | `runCommandExclusive` sets `commandInFlight = true` and awaits pane refresh **before** entering `try/finally`. If refresh throws, the lock is never cleared.                                            | All review actions can become permanently blocked for the session (user-visible deadlock until reload).                                  | Move the pre-refresh await inside `try/finally`, or wrap pre-refresh in its own `try/catch` that always resets `commandInFlight`.    | Unit test: mock `refreshCommentsPane` to throw once, call `runCommandExclusive` twice, assert second call still executes. |
| major    | `src/change-resolution.ts:69-74`                                                      | `rejectTrackedChangeByMarkup` uses `content.replace(match, '$1')` for deletion/substitution rejection. With string search, capture restoration is not performed, so old text is not restored correctly. | Rejecting a deletion/substitution via this path can silently remove content instead of restoring original text.                          | Extract capture groups with regex `exec` and replace using the extracted old text, or run a regex-based replace on the scoped match. | Add unit tests for deletion + substitution rejection: `a{--old--}b -> aoldb`, `a{~~old~>new~~}b -> aoldb`.                |
| major    | `tests/unit/main.unit.test.ts:4-17`, `tests/unit/change-resolution.unit.test.ts:8-40` | Tests currently do not exercise `runCommandExclusive` failure paths or `rejectTrackedChangeByMarkup` behavior.                                                                                          | High-risk regressions in concurrency and resolution code can ship unnoticed; this aligns with recent user-reported race/corruption bugs. | Add targeted tests for command lock release on exception and for all reject variants in `ChangeResolutionService`.                   | Run `npm test` after adding tests; ensure new tests fail on current buggy behavior and pass with fix.                     |
| minor    | `src/theme-presets.ts:9-10`, `src/theme-presets.ts:65-72`                             | Theme preset IDs are generated with `Date.now()` only. Fast consecutive saves can collide within the same millisecond.                                                                                  | Duplicate IDs can make later updates/deletes ambiguous and cause incorrect preset targeting.                                             | Use `crypto.randomUUID()` (or append monotonic counter/random suffix) and keep deterministic test injector via `IThemeIdGenerator`.  | Unit test with stub generator returning same value twice should assert collision handling (reject/regenerate).            |
