# Security Policy

## Supported Versions

Security fixes are applied to the latest released version.

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue first.

1. Open a private security advisory in this repository:
   - `Security` tab -> `Advisories` -> `Report a vulnerability`
2. Include:
   - affected version(s)
   - reproduction steps
   - expected vs. observed behavior
   - impact assessment

You can also contact the maintainer through GitHub: `@rohrbachd`.

## Disclosure Process

- Reports are acknowledged as soon as possible.
- Valid vulnerabilities are triaged and patched in a new release.
- Public disclosure happens after a fix is available.
